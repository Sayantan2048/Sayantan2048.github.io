I am a Computer Graphics student at McGill University. I joined McGill in September 2016 and before that I worked three years in a Power Plant working in various roles including instrumentation(primary), networking and security and web development. I completed my Bachelor's in 2013 in Electronics and Communication Engineering from National Institute of Technology, Durgapur, India. I had interest in graphics since I was an undergrad when I taught myself old OpenGL and enjoyed the EdX Course in Computer Graphics by Prof. Ramamoorthi. I also have this weird fascination for GPUs - probably because I couldn't play Crysis on my PC during my school days :P. So, I learned OpenCL and started contributing towards open source project John-The-Ripper where I built several GPU accelerated cryptography algorithms.

My current research:
  • Building friction model from microfacet distribution. This is also my Master's thesis. Since the origin of friction on meso-scale(order of mm) is due to the surface geometry which is also related to microfacet distribution used in path tracing, we are trying to build a model which behaves both physically and visually as per the surface geometry.
  • Effect of control variates in path tracing. We are exploring the effects of analytic and non-analytic integrators as control variate with the aim of reducing variance and improving convergence in path tracing and global illumination.

My other interests are:
  • Exploring Deep Learning and Reinforcement Learning methods for noise/variance reduction in path tracing and character animation.
  • GPU programming and shader development.

  3. Pathtracer

    4. I implemented this Pathtracer as part of Image Synthesis course. The tracer supports Explicit Path Tracing, BVH accelerator, BRDFs like GGX, Beckmann, Phong(Microfacet), Phong(Classic), BRDF sampling for all the mentioned BRDFs, Sphere and Area light sampling, Environment map sampling and Multiple Importance Sampling. I also implemented an analytic integrator based on Linearly Transformed Cosines paper.





  4. John-The-Ripper

    5. John-The-Ripper is an open-source password security auditing tool used by system administrators for recovering lost passwords. My main contributions were mask mode password generation on GPU and a lookup table- which requires exactly two memory access for checking cryptography hashes on GPU. My work speeds up the hash rate by a factor of 100x-1000x for fast hashes. Previously passwords were generated on CPU which had to be transferred to the GPU for processing and then fetched back from GPU for checking. Now both generation and checking are done on GPUs eliminating the latencies involved in the process.

Project: Implemented mask mode password generation on GPUs for John-The-Ripper security auditing tool. Generating data on GPU is much more crucial for fast cryptography hash types where data transfer overhead consumes majority of compute time.
